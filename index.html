How it works
1

Fork the repo

Head to github.com/modding-community-com/register and fork it to your account.

2

Create your JSON file

Add a file to the domains/ folder. The filename must match your subdomain exactly - cool-mod.json for cool-mod.modding-community.com. Use the builder below.

// domains/cool-mod.json { "subdomain": "cool-mod", "type": "CNAME", "target": "your-username.github.io" }
3

Open a Pull Request

Submit a PR from your fork to the main branch. Automated checks will run. A maintainer reviews and merges.

4

You're live

DNS goes active within minutes of the merge. Then configure your hosting provider to accept traffic from your new subdomain.

Rules

Check the repo README for the full details - this is just a quick summary.

One subdomain per target

Each target URL or IP can only map to one subdomain. No duplicate entries.

Filename = subdomain

The JSON filename must exactly match the subdomain value inside it.

Manual review

All PRs are reviewed by a maintainer. Offensive or impersonating names get rejected.

Keep it active

Subdomains pointing to dead or empty sites may be removed to free up names.

To report a subdomain: DM @atomictyler on Discord. To update or delete your subdomain, edit or remove your JSON file and open a new PR.

Hosting setup

After your PR is merged, tell your host about the new domain. Full instructions in the README.

  1. Go to your repo → Settings → Pages
  2. Under Custom Domain, enter your-subdomain.modding-community.com
  3. Click Save. GitHub verifies the DNS and issues an SSL cert automatically.
  1. Open your project dashboard on Vercel or Netlify
  2. Find Domain Settings or Custom Domains
  3. Add your-subdomain.modding-community.com
  4. The provider detects the CNAME automatically - no extra DNS step needed.

Use type A in your JSON with your server's public IPv4 as the target. Then configure your web server (Nginx, Apache, Caddy…) to handle the hostname your-subdomain.modding-community.com.

// domains/my-server.json { "subdomain": "my-server", "type": "A", "target": "123.456.789.0" }
